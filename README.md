
# Gym Management System

This project is developed as part of the Software Engineering Lab course.
It represents a Gym Management System designed and documented during weeks 1 to 9.

## 1. Project Description
The system manages gym members, subscriptions, class reservations, trainers, and messaging between trainers and members.

## 2. Functional Requirements
- Member registration
- Subscription management
- Class reservation
- Trainer management
- Messaging system
- Payment registration

## 3. Non-Functional Requirements
- Security and authentication
- Maintainability
- Scalability
- Clear system design

## 4. UML Diagrams
The following UML diagrams were designed during the project:
- Use Case Diagram
- Class Diagram
- Sequence Diagram
- Activity Diagram
- State Machine Diagram

## 5. System Architecture
The system follows a three-layer architecture:
- Front-end: User interface and interactions
- Back-end: Business logic and APIs
- Auth & Database: Authentication and data storage

## 6. Testing
Different types of tests were designed:
- Unit Tests
- Integration Tests
- System Tests

Test coverage was analyzed using coverage tools.
